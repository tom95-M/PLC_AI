# S7PlcSimulator 本地联调与测试说明

## 1. 启动模拟器
1. 在项目目录执行:
```powershell
dotnet run
```
2. 在界面中确认:
- `本地地址`: `0.0.0.0` (同机联调可改 `127.0.0.1`)
- `端口`: `1102` (避免 102 管理员权限问题)
3. 点击 `启动 Server`，状态应显示 `运行中`。

## 2. 变量准备
默认变量:
- `MotorRun`: `DB1.DBX0.0` (BOOL)
- `Counter`: `DB1.DBW2` (INT)
- `Pressure`: `DB1.DBD4` (REAL)

先在界面点击 `写入全部`，再点击 `读取全部`，确认日志无错误。

## 3. 客户端联调（S7 客户端）
你可以用任意 S7 客户端（如 S7.Net、Snap7 客户端、PLC 调试工具）连接:
- IP: `127.0.0.1`（同机）
- Port: `1102`
- Rack: `0`
- Slot: `1`

建议先做 3 个读写用例:
1. 写 `DB1.DBX0.0 = true`，界面点击 `读取选中`，`MotorRun` 应变为 `True`。
2. 写 `DB1.DBW2 = 321`，界面读取后 `Counter` 应为 `321`。
3. 写 `DB1.DBD4 = 12.5`，界面读取后 `Pressure` 应接近 `12.5`。

## 4. 性能与稳定性回归
1. 客户端高频轮询（例如 50-100ms）连续 2-5 分钟。
2. 观察界面日志:
- 现在会按 1 秒汇总显示 `[RW] 1s summary: READ=..., WRITE=...`，不再刷屏。
3. 观察 UI 是否卡顿、是否出现启动/停止异常。

## 5. 异常输入测试
在变量面板中验证输入校验:
1. `BOOL` 类型时将 `Bit` 改为 `8`，执行读写应报错。
2. `DB` 填 `0` 或 `Offset` 填负数，执行读写应报错。
3. `BOOL` 值支持 `true/false` 和 `1/0`。

## 6. 常见问题
- 连接失败:
  - 先确认状态是 `运行中`。
  - 端口与客户端配置一致（默认 `1102`）。
  - 跨机器联调时，客户端用服务器实际 IP，且放通防火墙入站端口。
- 102 端口失败:
  - Windows 通常需要管理员权限，建议继续用 `1102`。
